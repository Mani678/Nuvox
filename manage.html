🔐 Security Model
Nuvox uses a server-side verification architecture. Payments are verified by a Supabase Edge Function using your secret keys — which never touch the browser. The anon key is public by design; your RLS policies ensure only products can be read publicly. Orders are written exclusively by the Edge Function using the service role key. No customer data is accessible from the frontend.
📦 Bulk Import Products (CSV)
Upload a CSV file to import multiple products at once into your Supabase catalogue. Use this to quickly populate your store.
CSV format: name, brand, category, price, old_price, emoji, badge, description
🗄️ Supabase Setup Guide
Follow these steps to connect your admin panel and store to Supabase.
  • 1
    Go to supabase.com → New Project → name it "nuvox" → select West EU (London) region for best Nigeria latency
  • 2
    SQL Editor → New Query → paste and run supabase/production-setup.sql (the full production script, not the old dev one)
  • 3
    Project Settings → API → copy Project URL and anon public key → paste into both HTML files
  • 4
    Project Settings → API → copy service_role key → set as Edge Function secret (NOT in HTML files)
  • 5
    Deploy Edge Function: supabase functions deploy verify-payment then set secrets:
    supabase secrets set PAYSTACK_SECRET_KEY=sk_live_xxx
    supabase secrets set FLUTTERWAVE_SECRET_KEY=FLWSECK_TEST-xxx
    supabase secrets set SUPABASE_SERVICE_ROLE_KEY=eyJxxx
  • 6
    Change ADMIN_PASSWORD in this file to something strong before hosting it
